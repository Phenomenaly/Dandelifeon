# Dandelifeon Solver

This project is a high-performance solver designed to find optimal initial patterns for a specialized variation of [Conway's Game of Life](https://en.wikipedia.org/wiki/Conway%27s_Game_of_Life) implemented via the Dandelifeon flower in the Minecraft modification Botania. While standard cellular automata operate on infinite, boundary-free grids with static survival and birth rules, the Dandelifeon introduces several highly constrained modifications. The playing field is strictly bounded to a 25x25 grid, where cells are prohibited from breeding outside the borders. Furthermore, a 3x3 absorption zone sits at the center, housing the flower itself. If any active cell steps into this zone, the simulation immediately terminates, destroying all remaining cells on the board. The absorbed cells are converted into mana, a resource whose quantity is directly proportional to the cell's age—which increases by one on each step of its survival. Thus, the objective of the system is to design a starting cellular seed that lives as long as possible before entering the central zone in a highly synchronized wave of multiple cells on the final tick.

To navigate this highly chaotic and discontinuous fitness landscape, the solver employs a decentralized, multi-threaded optimization algorithm. Each CPU core runs an independent solver instance inside an isolated Lua state, utilizing an adaptive hill-climbing search with dynamic mutation weights. Instead of mutating walls randomly, the algorithm analyzes the physical footprint of the active cells' history. It applies a quadratic probability distribution to these visited coordinates, focusing mutations on walls further away from the starting figure's center of mass, as these distant obstacles primarily dictate the critical, late-stage steps of the simulation. Crucially, the search space is dramatically reduced by enforcing strict bilateral symmetry across the vertical center axis of the board. Because both the starting seed and the wall mutations are mirrored, the simulation remains perfectly symmetric, ensuring that cells enter the absorption zone in balanced pairs on the final tick. When a thread gets stuck in local stagnation, it temporarily increases its mutation rate before eventually retrieving a random elite pattern from a shared, thread-safe C++ repository of 1000 highly diverse solutions, thereby preventing premature convergence.

Beyond its practical application, this project was built with a clear academic and research-oriented objective, which naturally led to elements of deliberate over-engineering. The primary motivation was to study the low-level interoperability between C++ and Lua, particularly how to design a high-performance system where heavy computational lifting is done on the native side while complex optimization heuristics are written in flexible scripts. The integration was built using the raw, low-level Lua C-API, managing C++ object lifetimes inside the Lua garbage collector via the placement new operator and userdata wrappers to eliminate data copying overhead. Furthermore, the project serves as an exercise in clean architecture and zero-overhead abstractions. By decoupling the core SIMD simulation engine, the geometric transformation handlers, the thread-safe shared archives, the API binding layers, and the terminal user interface, the system remains modular, highly maintainable, and remarkably fast, executing hundreds of thousands of operations per second across multiple cores.

Ultimately, because of the chaotic nature of cellular automata where a single bit flip completely alters the long-term trajectory, the search is highly dependent on initial conditions and represents a form of computational gambling. Despite the structured constraints, symmetric search space reduction, and adaptive heuristics, the path to finding the perfect 36,000-mana pattern cannot be deterministically predicted. A single run of the program is subject to the luck of the draw: the absolute optimal pattern may be uncovered within ten minutes of launching the solver, or it may require a day or more of continuous, intensive computation on a multi-core server.

Looking forward, the roadmap for this project encompasses several key architectural and functional advancements. The primary objective is to scale the simulation engine to accommodate the expanded multi-field configuration, translating the current localized heuristics to a much larger 75x75 grid containing nine synchronized Dandelifeon fields. Alongside this physical scaling, a comprehensive restructuring of the C++ codebase is planned to achieve rigorous adherence to SOLID and clean architecture design principles, further decoupling the core mathematical services from the execution adapters. Additionally, the optimization parameters will be expanded beyond raw mana yields to encompass more complex operational constraints. This includes introducing multiobjective sorting that prioritizes space-minimizing patterns to reduce the physical footprint of the setup, as well as optimizing the initial placement of the cellular figures near the borders of adjacent fields, facilitating more elegant and compact automation layouts in a practical gameplay environment.
